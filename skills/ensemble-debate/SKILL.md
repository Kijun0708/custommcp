## Ensemble Debate

여러 전문가가 동일한 주제에 대해 독립적으로 분석한 후 토론합니다.

### 토론 프로세스

1. **독립 분석**: 각 전문가가 독립적으로 자신의 관점에서 분석
   - strategist: 아키텍처/전략적 관점
   - reviewer: 품질/보안 관점
   - researcher: 기술적 근거/레퍼런스 관점
2. **교차 검증**: 다른 전문가의 분석을 비판적으로 검토
3. **합의 도출**: 의견 차이를 논의하고 최종 합의안 도출

### 사용 시나리오

- 아키텍처 의사결정 (마이크로서비스 vs 모놀리스)
- 기술 스택 선택
- 성능 vs 유지보수성 트레이드오프
- 리팩토링 전략
- 보안 정책 수립

### 출력

- 각 전문가의 독립 분석
- 합의점과 의견 차이
- 최종 권장사항 (근거 포함)
